# NoteExam MVP 定义

> 版本：V1.0 | 状态：✅ 已完成 | 最后更新：2026-06-29

---

## 1. MVP 目标

### 核心假设

我们假设：**学习者愿意用 AI 将笔记转化为测试题，并能从中获得有效的学习反馈。**

MVP 的目标是用最小的功能集验证这一假设，收集真实用户反馈，为后续迭代提供方向。

### 验证指标

| 指标 | 衡量方式 |
|------|----------|
| **用户是否愿意上传笔记** | 上传转化率（访问 → 上传成功） |
| **生成的试卷是否被使用** | 生成 → 完成答题的留存率 |
| **用户是否认可出题质量** | 答题后满意度评分 / NPS |
| **用户是否愿意分享** | 分享率 |

## 2. MVP 功能范围

### ✅ 保留（MVP 必须实现）

| 模块 | 功能 | 理由 |
|------|------|------|
| **笔记上传** | 粘贴文本上传 | 零技术门槛，最快验证核心闭环，不依赖 OCR |
| **知识点提取** | AI 自动提取 + 结构化展示 | 核心价值所在，用户第一感知点 |
| **题型** | 单选题、填空题、判断题 | 三种最常用的客观题，AI 生成质量最稳定 |
| **难度** | 基础 + 进阶两档 | 覆盖主流水位，拔高题型可延后 |
| **答案** | 标准答案 + 简要解析 | 检验的闭环必须完整 |
| **试卷** | 生成 + 预览 | 完成从上传到查看的全流程 |

### ⏸ 延后（V1.1+）

| 模块 | 功能 | 延后理由 |
|------|------|----------|
| **笔记上传** | 图片上传（OCR） | 需接入 OCR 服务，增加技术复杂度和成本 |
| **笔记上传** | PDF 上传 | PDF 解析和 OCR 链路较长，延后处理 |
| **题型** | 多选题 | AI 生成多选题（尤其干扰项）质量不稳定，需更多调优 |
| **题型** | 简答题、论述题 | 主观题评分标准难统一，在线答题体验待打磨 |
| **难度** | 拔高难度 | 基础+进阶已覆盖多数场景 |
| **知识点** | 手动增删改 | 先验证自动提取质量，再补编辑能力 |
| **导出** | 试卷导出 | 在线答题可先跑通，导出格式（PDF/Word）选择待定 |

### MVP 功能矩阵

```
┌─────────────────┬──────────┬──────────┬──────────┬──────────┐
│                 │  单选题   │  填空题   │  判断题   │          │
├─────────────────┼──────────┼──────────┼──────────┼──────────┤
│   基础难度       │    ✅    │    ✅    │    ✅    │  选择题   │
├─────────────────┼──────────┼──────────┼──────────┤  填空题   │
│   进阶难度       │    ✅    │    ✅    │    ✅    │  判断题   │
└─────────────────┴──────────┴──────────┴──────────┴──────────┘
                   共 3 种题型 × 2 档难度 = 6 种组合
```

## 4. 上线标准

### 功能标准

- [x] 粘贴文本后，AI 能正确提取 ≥ 80% 的核心知识点
- [x] 生成的题目题意清晰、答案正确、解析准确（人工抽检通过率 ≥ 90%）
- [x] 试卷生成时间 ≤ 3 分钟（10 个知识点以内）
- [x] 支持移动端浏览器访问和操作

### 质量标准

- [x] 无阻断性 Bug，核心流程通畅
- [x] 生成失败时有明确提示和重试机制
- [x] 页面首屏加载 ≤ 3 秒

### 体验标准

- [x] 新用户无需教程即可完成首次出卷
- [x] 界面简洁清晰，无冗余元素

## 5. 版本迭代历史

| 版本 | 重点 | 内容 | 状态 |
|------|------|------|------|
| **V1.0（MVP）** | 核心闭环 | 粘贴文本、3 题型 + 2 难度、答案与解析、试卷预览 | ✅ 已完成 |
| **V1.1** | 拓宽输入 | 图片上传（OCR）、PDF 上传、多选题、拔高难度、知识点手动调整、试卷编辑、试卷导出 | ✅ 已完成 |
| **V1.2** | 丰富题型+闭环 | 简答题、论述题、自定义难度占比、历史试卷管理、错题回顾 | ✅ 已完成 |
| **V1.3** | App 落地 | PWA 封装（离线可用、可安装、桌面/移动端类原生体验） | ✅ 已完成 |
| **V1.4** | 在线答题+账号 | 在线答题（6 题型交互判分）、用户账号（注册/登录/数据隔离） | 🔨 当前迭代 |
| **V2.0** | 智能增强 | 学习报告、错题智能分析、微信小程序 | 📋 规划中 |

## 6. 不做的事情（明确边界）

- ❌ 不做题库社区或公共试题分享
- ❌ 不做在线考试监考/防作弊
- ❌ 不做语音输入
- ❌ 不做视频笔记分析
- ❌ MVP 阶段不做用户注册/登录（先以匿名使用验证核心价值）
