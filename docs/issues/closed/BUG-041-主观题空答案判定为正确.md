# BUG-041-主观题空答案判定为正确

## 基本信息

| 字段 | 值 |
|------|-----|
| BUG-ID | BUG-041 |
| 优先级 | P1 |
| 发现日期 | 2026-06-30 |
| 发现阶段 | 测试验收（v1.2） |
| 所属模块 | 前端 - 答题判分 |
| 影响范围 | 填空题、简答题、论述题的答题模式 |

## 问题描述

在试卷答题模式中，填空题（fill_blank）、简答题（short_answer）、论述题（essay）这类需要手动输入答案的题目，当输入框保持空白（什么也不输入），直接点"提交"按钮时，判分结果显示为 **正确**（绿色对勾）。

## 复现步骤

1. 点击任意试卷进入答题模式
2. 找到一道填空题/简答题/论述题
3. 输入框保持空白，不输入任何内容
4. 直接点击"提交"按钮
5. 观察判分结果

## 预期结果

空白答案应判定为 **错误**，不应显示"回答正确"。

## 实际结果

空白答案判定为 **正确**，显示绿色对勾。

## 根因分析

`ExamModal.jsx` 中的 `checkAnswer` 函数存在两处缺陷：

### 1. 填空题（fill_blank）：`String.includes('')` 始终为 true

```javascript
if (qType === 'fill_blank') {
    const ua = userAnswer.trim().replace(/\s+/g, '');
    const ca = correct.trim().replace(/\s+/g, '');
    return ua === ca || ua.includes(ca) || ca.includes(ua);
    //     ^^^^^^^^           ^^^^^^^^^^^^^^^^    ^^^^^^^^^^^^^^^^
    //     空串===非空串→false   空串.includes(ca)→false   ca.includes('')→TRUE!
}
```

当 `userAnswer` 为空字符串时，`ca.includes('')` 在 JavaScript 中始终返回 `true`。

### 2. 简答题/论述题（short_answer/essay）：无任何校验

```javascript
if (qType === 'fill_blank') { ... }
return true;  // short_answer 和 essay 直接落到这里，无条件返回正确
```

对 `short_answer` 和 `essay` 题型，函数未做任何判断，始终返回 `true`。

## 建议修复方案

1. 在函数开头增加空答案检查，任何题型空答案直接返回 `false`
2. 为 `short_answer` 和 `essay` 添加模糊匹配逻辑（或至少要求非空）
