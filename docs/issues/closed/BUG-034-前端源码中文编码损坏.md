# BUG-034: 前端源码中文编码损坏，全部变成"？"

**发现日期**: 2026-06-30
**优先级**: P0
**位置**: `app/frontend/src/pages/Home.jsx`, `app/frontend/src/components/ExamModal.jsx`, `app/frontend/src/api.js`
**状态**: 待修复

## 问题描述

研发部在编辑上述文件时（修复 BUG-031 相关），保存时的编码方式不正确，导致文件内所有中文字符被替换为 `?`（ASCII 0x3F）。

| 文件 | 损坏的 `?` 组数 |
|------|----------------|
| `pages/Home.jsx` | 41 处 |
| `components/ExamModal.jsx` | 18 处 |
| `api.js` | 1 处 |

## 影响

- 前端界面所有中文标签、Toast 提示、按钮文字全部显示为 `???` / `??`
- 用户完全无法使用（P0 阻断）

## 根因

编辑器/IDE 保存文件时使用了非 UTF-8 编码（如 ANSI/GBK），导致多字节中文字符被替换为 `?`，且原始中文内容已不可恢复。

## 修复建议

1. 从 Git 恢复原始文件：`git checkout -- app/frontend/src/pages/Home.jsx app/frontend/src/components/ExamModal.jsx app/frontend/src/api.js`
2. 确保编辑器使用 UTF-8 编码保存
3. 重新应用 BUG-031 的中文化修改（之前的中文已被编码损坏而丢失）
