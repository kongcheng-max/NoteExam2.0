# BUG-054: 注册表单缺少「确认密码」字段

**优先级**: P2 (UX 改进)  
**阶段**: V1.4  
**发现日期**: 2026-06-30  
**模块**: 用户认证  
**文件**: `app/frontend/src/App.jsx:48-64`、`app/backend/schemas.py:210-212`

## 问题描述

注册表单只有「邮箱」和「密码」两个输入框，缺少「确认密码」字段。用户输入密码时没有二次确认，容易因手误导致注册后无法登录。

## 期望行为

注册表单增加「确认密码」字段：
1. 前端增加 `confirmPassword` 输入框，提交前校验 `password === confirmPassword`
2. 后端 `UserRegister` schema 增加 `confirm_password: str` 字段，校验两次输入一致

## 修复建议

**前端 App.jsx：**
- 添加 `confirmPassword` state
- 注册模式下显示确认密码输入框
- 提交前校验两次输入一致

**后端 schemas.py：**
```python
class UserRegister(BaseModel):
    email: EmailStr
    password: str = Field(..., min_length=6)
    confirm_password: str = Field(..., min_length=6)

    @model_validator(mode='after')
    def check_passwords_match(self):
        if self.password != self.confirm_password:
            raise ValueError('两次输入的密码不一致')
        return self
```
