# BUG-053: 登录用户无法删除登录前创建的笔记/试卷

**优先级**: P1 (影响用户体验)  
**阶段**: V1.4  
**发现日期**: 2026-06-30  
**模块**: 后端权限校验  
**文件**: `app/backend/routers/notes.py:28-29`、`app/backend/routers/exams.py:410-411`

## 问题描述

用户在未登录状态下创建了笔记和试卷（`user_id = "default"`），登录后再尝试删除这些历史记录时，后端返回 403「无权修改公共笔记」/「无权删除公共试卷」。

根本原因：`_check_note_access` 和 exam delete 中对 `user_id == "default"` 的记录做了 `require_owner` 拦截，但登录用户应该有权限管理这些公开笔记。

## 复现步骤

1. 未登录状态下粘贴文本创建笔记，生成试卷
2. 注册并登录
3. 在历史列表中点击删除该笔记或试卷
4. 提示：「无权修改公共笔记」

## 期望行为

登录用户应能删除 `user_id == "default"` 的笔记和试卷。

## 修复建议

在 `notes.py:28-29` 和 `exams.py:410-411` 中，允许已登录用户（`user is not None`）操作 `default` 数据：

```python
# Before:
if require_owner and note.user_id == "default":
    raise HTTPException(status_code=403, detail="无权修改公共笔记")

# After:
if require_owner and note.user_id == "default" and not user:
    raise HTTPException(status_code=403, detail="无权修改公共笔记")
```

## 影响范围

- 笔记删除/编辑
- 试卷删除
