# BUG-055: 退出登录后仍可删除属于已登录用户的笔记和试卷

**优先级**: P1 (安全漏洞)  
**阶段**: V1.4  
**发现日期**: 2026-07-01  
**模块**: 后端权限校验  
**文件**: `app/backend/routers/notes.py:20-31`、`app/backend/routers/exams.py:398-411`

## 问题描述

用户登录后创建了笔记和试卷（`user_id = 真实用户ID`），退出登录后（无 Authorization header），仍可成功删除。两个端点的权限逻辑都有相同缺陷。

**notes.py `_check_note_access`：**
```python
# user 为 None → 条件跳过
if user and note.user_id != "default" and note.user_id != user.id:
if require_owner and note.user_id == "default" and not user:
if require_owner and user and note.user_id != "default" and note.user_id != user.id:
# 如果 note.user_id 不是 "default"，三条全部跳过 → 允许操作
```

**exams.py `delete_exam`：**
```python
# 同上逻辑
if note and user and note.user_id != "default" and note.user_id != user.id:
if note and note.user_id == "default" and not user:
# user 为 None 且 note 非 default → 全部跳过 → 允许删除
```

## 复现步骤

1. 登录 → 创建笔记 → 生成试卷
2. 退出登录（清除 token）
3. `DELETE /api/notes/{id}` → 200 成功
4. `DELETE /api/exams/{id}` → 200 成功

## 期望行为

未登录用户不能删除任何非公开数据。在删除入口增加：
```python
if not user:
    raise HTTPException(status_code=401, detail="请先登录")
```

对于笔记，应将此检查放在 `_check_note_access(require_owner=True)` 调用时。

## 影响范围

- 笔记删除（DELETE /api/notes/{id}）
- 笔记编辑（PUT /api/notes/{id}）
- 试卷删除（DELETE /api/exams/{id}）
