# BUG-042-用户账号缺少修改密码功能

## 基本信息

| 字段 | 值 |
|------|-----|
| BUG-ID | BUG-042 |
| 优先级 | P1 |
| 发现日期 | 2026-06-30 |
| 发现阶段 | 测试验收（v1.2 用户账号） |
| 所属模块 | 后端 + 前端 - 用户认证 |
| 影响范围 | 用户账号安全与体验 |

## 问题描述

feature-spec.md 第8节「用户账号」明确要求"可在设置中修改密码（需输入原密码）"，但当前代码完全没有密码修改功能。

## 缺失内容

### 后端
- `routers/auth.py` 没有 `PUT /api/auth/password` 或类似 endpoint
- 需要：接收旧密码 + 新密码，验证旧密码后更新 `password_hash`

### 前端
- 没有设置页面（Settings page）
- 没有密码修改 UI

## 预期行为

1. 用户点击头像 → 设置 → 修改密码
2. 输入原密码 + 新密码（至少6位）+ 确认新密码
3. 后端验证原密码正确后更新

## 建议

参考 feature-spec.md 第8节「退出与安全」：可在设置中修改密码（需输入原密码）
