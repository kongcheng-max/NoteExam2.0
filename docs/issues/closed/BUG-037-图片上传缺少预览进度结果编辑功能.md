# BUG-037: 图片上传缺少预览/进度/结果展示/编辑功能

**发现日期**: 2026-06-30
**优先级**: P1
**位置**: `app/frontend/src/pages/Home.jsx` 文件上传区域
**状态**: 待修复

## 问题描述

对照 `docs/v1.1-acceptance-checklist.md` 2.1 节，以下 4 项前端功能未实现：

| # | 缺失功能 | 当前状态 |
|---|---------|---------|
| 4 | 图片预览（缩略图+点击放大） | 无，选文件后只显示文件名 |
| 5 | OCR 识别进度（加载动画/进度提示） | 上传后无 OCR 处理状态反馈 |
| 6 | 识别结果展示（OCR 文本展示供校对） | OCR 完成后内容直接入库，用户看不到结果 |
| 7 | 识别文本可编辑（修改后再提交） | 无编辑入口，OCR 结果无法人工修正 |

## 影响

- 用户上传图片后不知道 OCR 是否在进行中
- OCR 识别错了也无法修正，直接影响后续试卷质量
- 体验割裂：上传 → 空白等待 → 突然出现知识点

## 修复建议

1. 选择文件后显示缩略图预览
2. 上传后轮询 `/api/notes/{id}` 检查 content 是否有内容来判断 OCR 完成
3. OCR 完成后将识别文本展示在可编辑的 textarea 中
4. 用户确认/修改后可提交更新 content
