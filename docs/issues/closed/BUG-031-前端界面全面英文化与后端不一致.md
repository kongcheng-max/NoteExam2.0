# BUG-031: 前端界面全面英文化，与中文后端不一致

**发现日期**: 2026-06-29
**优先级**: P1
**位置**: `app/frontend/src/` 全部组件
**状态**: 待修复

## 问题描述

V1.1 前端将所有用户界面文本和标签从中文改为了英文，与后端中文响应不一致：

| 位置 | 当前（英文） | 后端返回/应有（中文） |
|------|-------------|---------------------|
| 题型标签 | Single, Multi, T/F, Blank | 单选, 多选, 判断, 填空 |
| 难度标签 | Basic, Advanced, Hard | 基础, 进阶, 拔高 |
| Toast 提示 | Note uploaded, File uploaded... | 笔记上传成功, 文件上传成功... |
| 按钮文本 | Export, View, Delete, Edit, Del | 导出, 查看, 删除, 编辑, 删 |
| 空状态 | No notes yet | 暂无笔记 |
| 知识点管理 | Name is required, Updated, Deleted | 名称必填, 已更新, 已删除 |
| 试卷标题 | Exams, Q, Answer & Explanation | 试卷, 题, 答案与解析 |

涉及文件：`api.js`, `Home.jsx`, `ExamModal.jsx`

## 影响

- 后端返回中文错误提示 → 前端显示英文界面 → 用户体验割裂
- PRD 目标用户为中文用户
- 试卷内容（AI 生成）是中文，但框架标签是英文

## 修复建议

全局恢复中文标签。参考 MVP 阶段已验证的中文文案。
