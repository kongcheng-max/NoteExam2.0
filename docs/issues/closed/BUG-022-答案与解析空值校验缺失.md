# BUG-022: 答案与解析可能为空，校验缺失

**发现日期**: 2026-06-29
**优先级**: P1
**位置**: `app/backend/routers/exams.py:22-34` validate_question 函数
**状态**: 待修复

## 问题描述

手动测试发现简答题的答案和解析可能为空——试卷中显示"答案："后面无内容，解析也不显示。

根因：`validate_question`（BUG-004 修复引入）只校验了 `content.stem` 和选择题的 `options`，**完全遗漏了对 `answer` 和 `explanation` 字段的校验**。

```python
# routers/exams.py:22-34 当前实现
def validate_question(q: dict) -> bool:
    q_type = q.get("question_type", "")
    if q_type not in VALID_QUESTION_TYPES:
        return False
    if q.get("difficulty", "") not in VALID_DIFFICULTIES:
        return False
    content = q.get("content")
    if not isinstance(content, dict) or "stem" not in content:
        return False
    if q_type in CHOICE_TYPES:
        options = content.get("options")
        if not isinstance(options, list) or len(options) < 2:
            return False
    return True  # 没有校验 answer 和 explanation
```

## 影响

- 用户生成的试卷中部分题目没有答案，试卷不可用
- 前端 `ExamModal.jsx` 渲染 `{q.answer}` 为空字符串时只显示"答案："
- 影响所有题型，不仅是简答题

## 修复建议

在 `validate_question` 末尾增加答案和解析校验：

```python
def validate_question(q: dict) -> bool:
    # ... 现有校验 ...

    # 新增：答案不能为空
    answer = q.get("answer")
    if not answer or (isinstance(answer, str) and not answer.strip()):
        return False

    # 新增：解析不能为空（至少10个字符）
    explanation = q.get("explanation", "")
    if not explanation or len(str(explanation).strip()) < 10:
        return False

    return True
```

同时建议在 `services/prompts.py` 的 `QUESTION_GENERATE_PROMPT` 中强化要求，在第4条后增加：

```
4. 每题需包含：题干、选项（选择题）、标准答案、解析（1-3句话）
   **答案和解析绝对不能为空**
```
