# BUG-051: 填空题多空位（|| 分隔）判分不支持逐空比对

**优先级**: P1 (影响功能正确性)  
**阶段**: V1.4  
**发现日期**: 2026-06-30  
**模块**: 在线答题判分  
**文件**: `app/backend/routers/exams.py`（`_grade_question`）、`app/frontend/src/components/ExamModal.jsx`（`checkAnswer`）

## 问题描述

AI 生成填空题（`fill_blank`）答案时，对于多空位使用 `||` 分隔（例如：`"重要性均为low||具体定义和应用领域可能不同"`），但前后端判分逻辑都**不支持逐空比对**，仅将整个答案字符串作为整体比较。

当前判分逻辑：
```python
# 后端
ua_clean = ua.replace(" ", "")
ca_clean = correct.replace(" ", "")
return ua_clean == ca_clean or ua_clean in ca_clean
```

```js
// 前端
const uaClean = uaTrimmed.replace(/\s+/g, '');
const caClean = correct.trim().replace(/\s+/g, '');
return uaClean === caClean || uaClean.includes(caClean);
```

问题：
1. 多空位答案以 `||` 分隔，但判分未按空位拆分
2. 用户需要在单个输入框中用 `||` 连接所有答案才可能匹配（但 UI 没有任何提示）
3. 技术规格明确要求"支持多个空位逐一比对"

## 复现步骤

1. 生成一份包含填空题的试卷（知识点足够多时，AI 会生成多空位填空题）
2. 进入答题模式，对填空题输入正确答案
3. 观察判分结果：即使答案正确也会被判为错误

## 期望行为

多空位填空题应：
1. 按 `||` 拆分正确答案和用户答案
2. 逐空位比对（trim 后精确匹配或近似匹配）
3. 所有空位都正确才算答对

## 影响范围

- 填空题判分准确率（约占试卷 15-20% 的题目）
