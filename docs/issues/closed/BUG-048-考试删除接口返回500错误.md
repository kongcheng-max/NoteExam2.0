# BUG-048 - 考试删除接口返回500错误

## 严重程度：P1（影响核心功能）

## 发现时间
2026-06-30

## 复现步骤
1. 创建笔记并生成试卷
2. 调用 DELETE /api/exams/{exam_id} 删除试卷
3. 返回 500 Internal Server Error

## 预期结果
删除成功返回 {"success": true, "message": "试卷已删除"}

## 实际结果
返回 HTTP 500 错误

## 根因分析
routers/exams.py 中 delete_exam 函数使用了 select(Exam).join(Note) 但未加载 Note 关系。
在 async SQLAlchemy 中访问 exam.note 时无法进行懒加载，导致 MissingGreenlet 错误。

代码位置：app/backend/routers/exams.py delete_exam 函数
第 exam.note.user_id 等访问需要先通过 selectinload(Exam.note) 或 joinedload 预加载

## 影响范围
- 无论是否登录、是否为创建者，所有试卷删除操作均失败
- 历史试卷管理功能完全不可用

## 修改建议
方案1（推荐）：直接使用 exam.note_id 替代 exam.note.user_id，无需预加载 Note 关系
方案2：在查询中添加 options(selectinload(Exam.note))
