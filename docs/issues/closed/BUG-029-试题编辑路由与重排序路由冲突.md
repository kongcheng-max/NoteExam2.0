# BUG-029: 试题编辑路由与重排序路由冲突

**发现日期**: 2026-06-29
**优先级**: P1
**位置**: `app/backend/routers/exams.py` V1.1 路由注册顺序
**状态**: 待修复

## 问题描述

试题编辑路由和重排序路由存在路径冲突。FastAPI 按注册顺序匹配，`/{exam_id}/questions/{question_id}` 在 `/{exam_id}/questions/reorder` 之前注册，导致 "reorder" 被当作 `question_id` 匹配到编辑路由，重排序端点永远无法访问。

```python
# 注册顺序（问题所在）：
@router.put("/{exam_id}/questions/{question_id}")   # ← 先匹配，question_id="reorder"
async def update_question(...)

@router.put("/{exam_id}/questions/reorder")          # ← 永远匹配不到
async def reorder_questions(...)

@router.delete("/{exam_id}/questions/{question_id}")  # 无冲突
async def delete_question(...)
```

## 验证

```
PUT /api/exams/test123/questions/reorder → 404 {"detail":"试题不存在"}
                                        ↑ 被 update_question 捕获，查找 id="reorder" 失败
```

## 影响

- 试卷题目重排序功能完全不可用
- 用户调整题目顺序的请求会被当作编辑单题处理

## 修复建议

将 `reorder` 路由放在 `{question_id}` 路由之前注册：

```python
# reorder 必须在前
@router.put("/{exam_id}/questions/reorder")
async def reorder_questions(...)

@router.put("/{exam_id}/questions/{question_id}")
async def update_question(...)
```
