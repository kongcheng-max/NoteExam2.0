# BUG-044-用户登录态未关联业务数据

## 基本信息

| 字段 | 值 |
|------|-----|
| BUG-ID | BUG-044 |
| 优先级 | P1 |
| 发现日期 | 2026-06-30 |
| 发现阶段 | 测试验收（v1.2 用户账号） |
| 所属模块 | 后端 - 全业务路由 |
| 影响范围 | 笔记、试卷、错题本的数据隔离 |

## 问题描述

用户可以注册和登录，但登录状态完全不影响业务数据。所有笔记的 `user_id` 始终为 `default`，历史试卷和错题本对所有用户可见，登录/未登录用户体验无任何差异。

## 复现步骤

1. 注册账号 A → 登录 → 创建笔记 → 生成试卷
2. 退出登录
3. 仍能看到步骤1创建的笔记和试卷
4. 注册账号 B → 登录 → 也能看到账号A的数据

## 预期行为（参照 feature-spec.md）

| 功能 | 未登录 | 已登录 |
|------|--------|--------|
| 粘贴文本 → 出题 → 预览 | ✅ 可用 | ✅ 可用 |
| 图片/PDF 上传 | ✅ 可用 | ✅ 可用 |
| 在线答题 | ✅ 可用 | ✅ 可用 |
| 历史试卷保存 | ❌ 关闭页面后消失 | ✅ 持久保存 |
| 错题本 | ❌ 关闭页面后消失 | ✅ 持久保存 |
| 试卷导出 | ✅ 可用 | ✅ 可用 |

## 实际行为

所有数据对所有用户可见，登录态无隔离作用。

## 根因分析

1. `notes.py`、`exams.py`、`wrong_answers.py` 的查询未按 `user_id` 过滤
2. 创建笔记时 `user_id` 硬编码为 `default`，未使用当前登录用户ID
3. 各路由未注入 `get_current_user` 依赖

## 建议修复

1. 在各业务路由中注入 `get_current_user` 依赖
2. 创建资源时写入当前用户 ID
3. 查询资源时按 `user_id` 过滤（或 `user_id = 'default'` 为公共数据）
