# BUG-056 学习报告生成接口返回500错误

## 基本信息

| 字段 | 内容 |
|------|------|
| **编号** | BUG-056 |
| **发现版本** | V2.0 |
| **严重程度** | P0（阻塞核心功能） |
| **发现日期** | 2026-07-01 |
| **状态** | 待修复 |

## 问题描述

调用 POST /api/reports/generate 生成学习报告时，后端返回 HTTP 500 Internal Server Error，无任何错误详情。整个学习报告功能不可用。

## 复现步骤

1. 注册用户并登录
2. 上传笔记，提取知识点，生成试卷
3. 提交试卷答题（产生答题记录和错题记录）
4. 调用 POST /api/reports/generate（不传或传入 exam_ids）
5. 后端返回 500

## 根因分析

pp/backend/routers/reports.py 第 60-83 行的错题查询：

`python
wa_query = select(WrongAnswer).join(...).where(...)
wa_result = await db.execute(wa_query)
wrong_answers = wa_result.scalars().all()

for wa in wrong_answers:
    wrong_data.append({
        "question_type": wa.question.question_type if wa.question else "",
        ...
    })
`

访问 wa.question 时触发 SQLAlchemy 懒加载，但 scalars().all() 返回的对象已脱离 async greenlet 上下文，导致 sqlalchemy.exc.MissingGreenlet 异常。

**对比**：pp/backend/routers/wrong_answers.py 中类似的查询正确使用了 .options(selectinload(WrongAnswer.question))，工作正常。

## 预期行为

POST /api/reports/generate 应正常生成并返回学习报告。

## 修复建议

在 eports.py 的 wa_query 中添加 .options(selectinload(WrongAnswer.question))：

`python
# reports.py 第 60 行附近
wa_query = select(WrongAnswer).join(...).where(...).options(selectinload(WrongAnswer.question))
`

同时确认 selectinload 已在文件顶部导入：
`python
from sqlalchemy.orm import selectinload
`

## 影响范围

- POST /api/reports/generate 完全不可用
- 学习报告模块核心功能阻塞
