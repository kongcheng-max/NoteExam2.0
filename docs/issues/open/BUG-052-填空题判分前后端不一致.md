# BUG-052: 填空题判分前后端逻辑不一致（包含方向相反）

**优先级**: P2 (影响判分一致性)  
**阶段**: V1.4  
**发现日期**: 2026-06-30  
**模块**: 在线答题判分  
**文件**: `app/backend/routers/exams.py:536-539`、`app/frontend/src/components/ExamModal.jsx:71-75`

## 问题描述

填空题的近似匹配逻辑在前端和后端实现**方向相反**，导致同一答案可能在前端显示正确，提交到后端后却被判错误（或反之）。

**后端逻辑** (exams.py `_grade_question`):
```python
return ua_clean == ca_clean or ua_clean in ca_clean
# 含义：用户答案 == 标准答案 OR 用户答案是标准答案的子串
```

**前端逻辑** (ExamModal.jsx `checkAnswer`):
```js
return uaClean === caClean || uaClean.includes(caClean);
// 含义：用户答案 == 标准答案 OR 用户答案包含标准答案
```

## 不一致示例

| 标准答案 | 用户输入 | 前端结果 | 后端结果 | 不一致？ |
|----------|----------|----------|----------|----------|
| F=ma | F=ma 补充文字 | ✅ 正确 | ❌ 错误 | ⚠️ 是 |
| 牛顿第二定律 F=ma 更多 | F=ma | ❌ 错误 | ✅ 正确 | ⚠️ 是 |

## 期望行为

前后端判分逻辑必须**完全一致**。建议统一为后端逻辑（`ua_clean in ca_clean`），即用户答案可以是标准答案的子串（允许只答关键部分），但反过来不行（不允许用户答案包含大量无关文字）。

## 影响范围

- 填空题判分结果前后端不一致
- 用户在前端看到"正确"，提交后可能被判错
