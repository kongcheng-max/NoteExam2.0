# BUG-043-注册接口未校验邮箱格式

## 基本信息

| 字段 | 值 |
|------|-----|
| BUG-ID | BUG-043 |
| 优先级 | P2 |
| 发现日期 | 2026-06-30 |
| 发现阶段 | 测试验收（v1.2 用户账号） |
| 所属模块 | 后端 - 用户认证 |
| 影响范围 | 注册数据质量 |

## 问题描述

注册接口 `POST /api/auth/register` 接受任意格式的邮箱，包括不含 "@" 的字符串。

## 复现步骤

```powershell
Invoke-RestMethod http://localhost:8000/api/auth/register -Method Post -Body '{"email":"no-at-sign-here","password":"123456"}'
```

## 实际结果

返回 `success: True`，用户创建成功，email 为 `no-at-sign-here`。

## 预期结果

应返回 422 校验错误，拒绝非法邮箱格式。

## 根因分析

`schemas.py` 中 `UserRegister.email` 字段仅使用 `str` + `min_length=5`，未使用 `EmailStr` 或正则校验：

```python
class UserRegister(BaseModel):
    email: str = Field(..., min_length=5, max_length=200, description="邮箱")
```

## 建议修复

使用 `pydantic` 的 `EmailStr` 类型，或添加正则校验 `pattern`。
