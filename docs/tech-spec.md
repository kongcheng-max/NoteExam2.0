## 1. 系统架构

NoteExam 采用前后端分离的三层架构，核心 AI 能力由 DeepSeek 大模型提供。

```
┌──────────────────────────────────────────────────────────┐
│                        用户端                             │
│  ┌──────────────────────────────────────────────────┐    │
│  │            Web 前端 (React / Vue)             │    │
│  │    笔记上传 · 知识点展示 · 试卷预览 · 在线答题      │    │
│  └───────────────────┬──────────────────────────────┘    │
└──────────────────────┼───────────────────────────────────┘
                       │ HTTPS
┌──────────────────────┼───────────────────────────────────┐
│                后端服务层                                  │
│  ┌───────────────────┴──────────────────────────────┐    │
│  │            API 网关 (Nginx / 云网关)              │    │
│  └───────┬──────────────────────────────┬───────────┘    │
│          │                              │                │
│  ┌───────┴───────────┐    ┌────────────┴────────────┐   │
│  │   核心业务服务      │    │   AI 调度服务            │   │
│  │  (Python FastAPI)  │    │  (Python FastAPI)       │   │
│  │                   │    │                         │   │
│  │  · 用户管理        │    │  · 知识提取 Prompt 编排  │   │
│  │  · 笔记处理        │    │  · 试题生成 Prompt 编排  │   │
│  │  · 试卷管理        │    │  · 答案生成 Prompt 编排  │   │
│  └───────┬───────────┘    └────────────┬────────────┘   │
└──────────┼─────────────────────────────┼────────────────┘
           │                             │
    ┌──────┴──────┐              ┌───────┴────────┐
    │  PostgreSQL │              │  DeepSeek API  │
    │   (数据库)   │              │  (大模型服务)   │
    └─────────────┘              └────────────────┘
```

### 架构说明

- **前端**：负责用户交互，所有 AI 调用通过后端中转，前端不直接调用 DeepSeek API；框架可选 React 或 Vue
- **核心业务服务**：处理笔记存储、试卷 CRUD、用户管理等业务逻辑
- **AI 调度服务**：封装与 DeepSeek 的交互，管理 Prompt 模板、Token 控制、错误重试
- **数据库**：存储用户数据、笔记内容、知识点、试卷、答案

## 2. 技术选型

### 选型原则

1. **API 优先使用国内厂商**，降低网络延迟和政策风险
2. **同一厂商优先**，减少供应商管理成本
3. **开源优先**，降低许可证成本

### 前端

| 选型 | 方案 | 理由 |
|------|------|------|
| 框架 | **React (Next.js) / Vue (Nuxt)** | React 生态成熟，Vue 上手更友好；均支持 SSR |
| UI 组件库 | **shadcn/ui / Ant Design Vue** | 分别对应 React 和 Vue 生态的主流组件库 |
| 状态管理 | **Context+SWR / Pinia** | MVP 阶段轻量方案即可 |
| 部署 | **阿里云 ECS / 腾讯云轻量服务器** | 国内云服务商，备案便捷 |

### 后端

| 选型 | 方案 | 理由 |
|------|------|------|
| 语言 & 框架 | **Python + FastAPI** | AI 生态最好，DeepSeek 官方 SDK 支持；高性能异步框架 |
| API 文档 | **自动生成 OpenAPI (Swagger)** | FastAPI 内置，零额外成本 |
| 数据库 | **PostgreSQL** | 成熟的关系型数据库，支持 JSON 字段适合存储试卷结构 |
| 数据库托管 | **阿里云 RDS / 腾讯云 PostgreSQL** | 国内云厂商，运维托管 |
| 缓存 | **Redis**（云服务商托管） | 用于 Prompt 缓存、会话管理 |

### AI 服务

| 选型 | 方案 | 理由 |
|------|------|------|
| 大语言模型 | **DeepSeek API**（深度求索） | 国内头部大模型，中文理解能力强，性价比高 |
| OCR 识别（V1.1） | **百度智能云 OCR** | DeepSeek 暂无 OCR 产品；百度 OCR 是国内最成熟的文字识别服务，识别准确率高 |

> **关于"同一厂商优先"原则**：核心模型 DeepSeek 来自深度求索，但其产品线目前不含 OCR 和云基础设施服务。因此 OCR 推荐百度智能云（国内 OCR 市场份额第一），数据库与部署推荐阿里云或腾讯云，均为国内头部服务商。

### 存储

| 选型 | 方案 | 理由 |
|------|------|------|
| 文件存储 | **阿里云 OSS / 腾讯云 COS** | 用于存储用户上传的图片、PDF |
| 文本存储 | **PostgreSQL** | 笔记文本、试卷内容直接存数据库 |

## 3. DeepSeek 集成

### 3.1 模型选择

| 场景 | 推荐模型 | 理由 |
|------|----------|------|
| 知识点提取 | DeepSeek-V3 | 长文本理解能力强，适合分析整篇笔记 |
| 试题生成 | DeepSeek-V3 | 指令遵循能力强，出题格式稳定 |
| 答案生成 | DeepSeek-V3 | 与出题使用同一模型，保持风格一致 |
| 轻量任务（如格式校验） | DeepSeek-V2.5 或轻量模型 | 降低延迟和成本 |

### 3.2 调用流程

```
用户上传笔记
     │
     ▼
┌─────────────┐
│ 文本预处理    │  ← 去除噪声、分段、Token 估算
└──────┬──────┘
       │
       ▼
┌─────────────────┐
│ Prompt 1: 知识提取 │  ← 输入：笔记全文 + 知识提取指令
│ → DeepSeek API    │  ← 输出：结构化知识点列表 (JSON)
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│ Prompt 2: 试题生成 │  ← 输入：知识点列表 + 题型 + 难度 + 数量
│ → DeepSeek API    │  ← 输出：试卷 (JSON，含题干/选项/答案/解析)
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│ 格式校验 & 组装   │  ← 校验 JSON 结构、拼装最终试卷
└─────────────────┘
```

### 3.3 Prompt 策略

- **知识提取 Prompt**：要求模型以 JSON 格式输出，包含知识点名称、类别、简要描述
- **试题生成 Prompt**：明确指定题型、难度、数量、输出格式；要求题干清晰、选项互斥、答案准确
- **解析生成 Prompt**：要求 1–3 句话，点明核心原理或常见误区

### 3.4 Token 估算

| 环节 | 输入 Token | 输出 Token | 备注 |
|------|-----------|------------|------|
| 知识提取 | ~2,000–8,000 | ~500–2,000 | 取决于笔记长度 |
| 试题生成 | ~1,000–3,000 | ~2,000–8,000 | 取决于题目数量和题型 |
| 合计 | ~3,000–11,000 | ~2,500–10,000 | 单次生成约消耗 10K–20K Token |

## 4. 数据流

### 4.1 核心数据流

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  笔记文本  │───▶│ 知识提取   │───▶│ 试题生成   │───▶│ 格式组装   │───▶│ 试卷展示   │
│  (上传)    │    │ (AI)      │    │ (AI)      │    │ (后端)    │    │ (前端)    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
      │               │               │               │               │
      ▼               ▼               ▼               ▼               ▼
  笔记存储到       知识点存储到     试题存储到      试卷存储到      用户浏览/
  数据库           数据库          数据库          数据库          答题
```

### 4.2 核心数据表结构（概要）

```
notes (笔记表)
  ├── id, content, type(text/image/pdf), user_id, created_at

knowledge_points (知识点表)
  ├── id, note_id, name, category, importance, created_at

exams (试卷表)
  ├── id, note_id, title, config(json), created_at

questions (试题表)
  ├── id, exam_id, type, difficulty, content(json), answer, explanation, order
```

## 5. 部署方案

### 推荐方案：国内云服务器 + 对象存储

| 组件 | 部署方式 | 推荐服务商 |
|------|----------|------------|
| 前端 | 静态资源托管 / Nginx | 阿里云 ECS + CDN 或 腾讯云轻量服务器 |
| 后端 | Docker 容器化部署 | 阿里云 ECS / 腾讯云 CVM |
| 数据库 | 云托管 PostgreSQL | 阿里云 RDS / 腾讯云 PostgreSQL |
| 文件存储 | 对象存储 | 阿里云 OSS / 腾讯云 COS |
| 域名 & HTTPS | 云服务商一站式 | 阿里云 / 腾讯云 |

### 环境要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 后端服务器 | 2 vCPU / 4 GB RAM | 4 vCPU / 8 GB RAM |
| 数据库 | 2 vCPU / 4 GB RAM / 50 GB SSD | 4 vCPU / 8 GB RAM / 100 GB SSD |
| 前端托管 | 静态托管即可 | CDN 加速 |

## 6. 安全与隐私

### 数据传输

- 全链路 HTTPS 加密
- API 密钥（DeepSeek API Key）仅存储在服务端，前端不可见
- 数据库连接使用 SSL

### 数据存储

- 用户笔记内容加密存储
- 数据库访问权限最小化，按服务划分账号
- 定期自动备份数据库

### 隐私保护

- 用户上传的笔记内容**不用于**模型训练（在 DeepSeek API 调用中设置 `do_not_train` 参数）
- 用户可随时删除自己的笔记和生成的试卷
- 不收集不必要的用户个人信息

### 内容安全

- 输入侧：对用户上传的文本内容进行合规审核（接入国内内容安全 API）
- 输出侧：对 AI 生成的试卷内容进行敏感词过滤

## 7. 容错与监控

### 容错策略

| 场景 | 策略 |
|------|------|
| DeepSeek API 超时 | 自动重试 2 次，超出后返回友好提示 |
| DeepSeek API 限流 | 排队机制，提示用户预估等待时间 |
| 生成结果格式异常 | JSON 校验 + 回退重试，失败后人工兜底提示 |
| 数据库异常 | 主备切换（云服务商托管自动处理） |

### 监控指标

| 指标 | 说明 |
|------|------|
| API 调用成功率 | DeepSeek API 调用成功 / 总调用 |
| 平均生成时间 | 从提交到试卷展示的端到端耗时 |
| 格式校验通过率 | 生成结果一次通过 JSON 校验的比例 |
| 用户满意度 | 答题后评分 |
