# NoteExam 产品需求文档（PRD）

> 版本：V1.2 | 状态：进行中 | 最后更新：2026-06-30

---

## 1. 产品背景

教育场景中，出题一直是耗时最多、门槛最高的环节之一。教师需要花费大量时间设计试题；学生学完知识后缺乏即时的自测手段；自学者更是难以评估学习效果。

NoteExam 利用 DeepSeek 大模型的自然语言理解能力，将学习笔记自动转化为覆盖知识点的标准试卷，让出题从「小时级」缩短到「分钟级」。

---

## 2. 目标用户

### 2.1 学生

- **场景**：期末考试、考研、考证备考
- **痛点**：手头笔记零散，不知道重点；想自测但没有针对性试卷
- **期望**：上传笔记 → 立即获得覆盖所有知识点的测试题 → 做完即看答案和解析

### 2.2 教师

- **场景**：课堂小测、单元测试、期中/期末出卷
- **痛点**：手动出题耗时、题型编排费脑、难以兼顾不同水平学生
- **期望**：上传课件/教材 → 自动生成多难度层级的试卷 → 稍作调整即可使用

### 2.3 自学者

- **场景**：自学编程、语言、考证等
- **痛点**：学完没有检验手段，不知道自己是否掌握
- **期望**：粘贴笔记 → 生成自测试卷 → 及时发现薄弱点

## 3. 功能需求

### 3.1 笔记上传（P0）

| 编号 | 功能点 | 用户故事 | 验收标准 |
|------|--------|----------|----------|
| F1.1 | 粘贴文本上传 | 作为学习者，我想把笔记文字直接粘贴进去，以便最快速度生成试卷。 | 支持纯文本粘贴，字数无硬限制；粘贴后即时展示文本预览。 |
| F1.2 | 图片上传 | 作为学习者，我想拍照上传手写笔记或课本截图。 | 支持 JPG/PNG 格式；单次上传多张图片；自动 OCR 识别。 |
| F1.3 | PDF 上传 | 作为学习者，我想直接上传 PDF 课件或扫描版教材。 | 支持 PDF 格式；自动提取文本内容；单文件建议 ≤ 50 页。 |

### 3.2 知识点提取（P0）

| 编号 | 功能点 | 用户故事 | 验收标准 |
|------|--------|----------|----------|
| F2.1 | AI 自动提取 | 作为学习者，我不想手动整理知识点，希望 AI 自动完成。 | 上传后自动生成结构化知识点清单；每个知识点含名称和类别。 |
| F2.2 | 知识点展示 | 作为学习者，我想浏览 AI 提取的知识点列表，了解覆盖范围。 | 以列表/卡片形式展示知识点；显示类别和重要度标签。 |
| F2.3 | 手动调整 | 作为学习者，我想增删改知识点，以控制出题范围。 | 支持删除、合并、新增知识点；调整后实时更新列表。 |

### 3.3 题型支持（P0）

| 编号 | 功能点 | 用户故事 | 验收标准 |
|------|--------|----------|----------|
| F3.1 | 单选题 | 作为学习者，我需要选择题来快速检验概念掌握。 | 每题 4 个选项，仅 1 个正确答案；题干与选项含义清晰。 |
| F3.2 | 多选题 | 作为学习者，我需要多选来检验综合判断能力。 | 每题 4–6 个选项，2 个及以上正确答案。 |
| F3.3 | 填空题 | 作为学习者，我需要填空来检验术语和公式记忆。 | 题干中含 1–3 处空白；答案明确无歧义。 |
| F3.4 | 判断题 | 作为学习者，我需要判断来快速检验基本概念。 | 陈述句命题，答案明确为"正确"或"错误"。 |
| F3.5 | 简答题 | 作为学习者，我需要简答来检验理解与表达能力。 | 题目明确提问方向；答案要点 2–4 条。 |
| F3.6 | 论述题 | 作为学习者，我需要论述题来检验综合分析和表达能力。 | 题目有明确论述方向；答案要点覆盖主要得分点。 |

### 3.4 难度分层（P0）

| 编号 | 功能点 | 用户故事 | 验收标准 |
|------|--------|----------|----------|
| F4.1 | 基础难度 | 作为初学者，我需要基础题来巩固核心概念。 | 题目聚焦定义、公式、基础事实；不涉及多步骤推理。 |
| F4.2 | 进阶难度 | 作为中等水平学习者，我需要进阶题来检验理解与应用。 | 题目涉及综合应用、易混淆辨析、多步骤推理。 |
| F4.3 | 拔高难度 | 作为冲刺高分的学习者，我需要拔高题来挑战综合能力。 | 题目涉及跨知识点融合、复杂分析、创新推理。 |
| F4.4 | 难度占比调整 | 作为用户，我想自定义三档难度的比例。 | 支持滑块或预设模版（基础巩固/综合训练/冲刺拔高）调整占比。 |

### 3.5 答案与解析（P0）

| 编号 | 功能点 | 用户故事 | 验收标准 |
|------|--------|----------|----------|
| F5.1 | 标准答案 | 作为学习者，我需要每道题有明确答案。 | 每题标注正确答案；多选标注所有应选项；简答/论述标注要点。 |
| F5.2 | 简要解析 | 作为学习者，我需要知道为什么对、为什么错。 | 解析 1–3 句话；指出核心原理或常见误区；关联对应知识点。 |

### 3.6 试卷管理（P0）

| 编号 | 功能点 | 用户故事 | 验收标准 |
|------|--------|----------|----------|
| F6.1 | 试卷生成 | 作为学习者，点击生成后能在合理时间内获得试卷。 | 生成时间 ≤ 3 分钟（常规内容）；生成中显示进度提示。 |
| F6.2 | 试卷预览 | 作为学习者，我想在界面上浏览整份试卷。 | 逐题展示，支持翻页浏览；显示题目序号、题型标签、难度标签。 |
| F6.3 | 试卷编辑 | 作为教师，我想对生成的题目进行微调。 | 支持编辑题干、选项、答案；支持调整题目顺序；支持删除单题。 |
| F6.4 | 试卷导出 | 作为教师，我想导出试卷用于打印或分发。 | 支持导出为可打印格式；保留排版和难度标识。 |

## 4. 非功能性需求

| 类别 | 需求 | 指标 |
|------|------|------|
| 性能 | 试卷生成速度 | 常规笔记（≤ 10 个知识点）生成时间 ≤ 3 分钟 |
| 性能 | 页面加载速度 | 首屏加载 ≤ 3 秒 |
| 可用性 | 操作门槛 | 新用户无需教程即可完成首次出卷 |
| 可用性 | 错误处理 | 上传失败、生成失败时给出明确提示和重试入口 |
| 兼容性 | 浏览器 | 支持 Chrome / Edge / Safari 主流版本 |
| 兼容性 | 移动端 | 响应式设计，手机端可完成上传和答题 |
| 安全 | 数据传输 | 全链路 HTTPS 加密 |
| 安全 | 用户数据 | 笔记内容不用于模型训练，可手动删除 |

## 5. 优先级分级

| 优先级 | 范围 | 对应版本 |
|--------|------|----------|
| **P0** | 粘贴文本上传、知识点提取（自动+展示）、单选题+填空题+判断题、基础+进阶难度、答案+解析、试卷生成与预览 | ✅ 已完成（MVP） |
| **P1** | 图片上传（OCR）、PDF 上传、多选题、拔高难度、知识点手动调整、试卷编辑、试卷导出 | ✅ 已完成（V1.1） |
| **P2** | 简答题、论述题、自定义难度占比、历史试卷管理、错题回顾、用户账号 | 🔨 当前迭代（V1.2） |

## 6. 约束与风险

| 类型 | 内容 |
|------|------|
| 技术约束 | API 优先使用国内厂商服务，同一厂商优先；DeepSeek 作为核心模型 |
| 模型依赖 | 出题质量强依赖 DeepSeek 模型能力，需持续关注模型版本和效果 |
| OCR 准确率 | 手写笔记和扫描件的 OCR 准确率影响知识点提取质量 |
| 内容安全 | 需对用户上传内容和生成内容进行合规审核 |
